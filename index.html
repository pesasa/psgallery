Psgallery examples

Here are couple of examples, how to use psgallery.js.

The gallery is written as html-list, which appears as html code as this:

<ul class="psgallery1"> <li><a href="images/DSC_0818.JPG">Redcurrant berries</a></li> <li><a href="images/DSC_0995.JPG">Flowers</a></li> <li><a href="images/DSC_1001.JPG">Flower and insect</a></li> <li><a href="images/IMAG0129.jpg">Autumn has come.</a></li> <li><a href="images/p4260052b.jpg">Spring time in a park.</a></li> <li><a href="images/p4260055b.jpg">By the sea</a></li> <li><a href="images/p4260057b.jpg">By the sea 2</a></li> <li><a href="images/p4260134.jpg">Squirrel in the park eating some nuts.</a></li> </ul>

And normally a browser renders it like this:

psgallery.js searches for thumbnails with given filename pattern and shows them and makes them clickable. Clicking of a thumbnail opens the larger image in overlay view mode which can be browsed forwards and backwards.

Navigation can be done either with mouse or with keyboard. Left and right arrows browse the images, f-key toggles fullscreen mode and esc quits the overlay view mode.

With thumbnail images

Here there are separate thumbnail images for each picture. By default psgallery searches for picture named filename.jpg a thumbnail named filename-tn.jpg. The added suffix can be changed by using option tnadd.

For example:

jQuery('.psgallery1').psgallery({ tnadd: '-tn' });

Without thumbnail image files

This is strongly discouraged! This can be slow and bad.

If there are no separate thumbnail images, big images can be used as thumbnails. By default this is not used, since using large image files as thumbnails is heavy! This can be turned on by setting option notn: true.

The maximum (pixel) size of the thumbnail can be given as option tnmaxsize for both width and height together or with options tnmaxwidth and tnmaxheight separately.

For example:

jQuery('.psgallery2').psgallery({ notn: true, tnmaxsize: 150 }); // or jQuery('.psgallery2').psgallery({ notn: true, tnmaxheight: 200, tnmaxwidth: 500 });

Other options

By default the gallery is not shown in full screen mode. Full screen mode can be turned on with fullscreen: true.

By default next and previous buttons work circularly, i.e., next of the last picture is the first one and the previous of the first image is the last one. This can be turned of with option circular: false. This removes previous and next buttons from first and last image respectively.

You can add a thumbnail preview on top of the view mode with option filmstrip: true. This is off by default.

The user can choose an effect for changing the images with option effect. Available effects are fade, resize and flip. By default, the gallery uses fading. Effects are done with CSS rules and one can "easily" make own effects. See the source code for examples.

For example:

jQuery('.psgallery3').psgallery({ fullscreen: true, circular: false, tnmaxheight: 100, tnmaxwidth: 500, filmstrip: true, effect: 'flip' });

Own style

The default outlook of the galleri is built in the psgallery.js. It is possible, however, to customize the user interface with own css-file.

The option style: "ownclass"' adds html-class "ownclass" to the wrapper element of gallery and this class can be used to override default css-rules in your own css-file.

For example (see the css-file psg_blue.css):

jQuery('.psgallery4').psgallery({ filmstrip: true, style: "psg_blue" });
